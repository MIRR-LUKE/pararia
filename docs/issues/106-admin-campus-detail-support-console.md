# 校舎詳細の read-only support console を実装する

## 状態

- Done
- GitHub Issue: `#209`
- 作成日: `2026-04-26`

## 目的

運営者が校舎詳細を開いたときに、本文や音声を見なくても、校舎状態・要対応理由・ユーザー/ジョブ/端末の概況を把握できるようにする。

## 背景

何百校舎を管理するには、校舎単位の drilldown が必要。ただし初回は read-only に限定し、危険操作や PII アクセスは後続の監査フレームワーク後に入れる。

## やること

- `/admin/campuses/[organizationId]` 相当の詳細画面を追加する
- 上部に `校舎名`, `状態`, `担当CS`, `最終利用`, `未対応件数` を固定表示する
- タブを `概要`, `ユーザー`, `ジョブ`, `端末`, `監査`, `メモ` に限定する
- 生徒/面談は件数と状態のみを初期表示する
- 面談本文・音声・保護者連絡先は表示しない
- 最近の監査ログを read-only で表示する

## 完了条件

- 運営者は校舎詳細で一次対応に必要な状況を把握できる
- PII や面談本文なしで要対応理由を説明できる
- 危険操作が常時露出していない
- 校舎詳細 API は対象校舎を明示して取得する
