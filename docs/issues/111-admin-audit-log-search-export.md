# 監査ログ検索・エクスポートを実装する

## 状態

- Deferred
- GitHub Issue: `#214`
- 作成日: `2026-04-26`

## 目的

誰が、いつ、どの校舎に、何をしたかを運営側で説明できるようにする。

## 背景

上場企業に見せられるSaaS品質にするには、監査ログが「DBにある」だけでは足りない。検索、絞り込み、重要操作の詳細、権限つきエクスポートが必要。

## やること

- `/admin/audit` 相当の監査検索画面を追加する
- 初期表示は検索フォームと最近の重要操作 20 件にする
- 検索軸は `期間`, `操作者`, `校舎`, `操作種別`, `結果` に絞る
- IP / targetId / request metadata は詳細検索に置く
- 重要操作詳細で理由、変更前後、承認者、関連チケットを表示する
- CSV / JSON export は権限限定にし、export 自体も監査する

## 完了条件

- 運営者が監査ログから説明責任を果たせる
- export は権限と監査つきで実行される
- 初期画面がログの大海になっていない
- 重要操作の理由と対象が追える
