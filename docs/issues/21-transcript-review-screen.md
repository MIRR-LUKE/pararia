# UI: 文字起こし確認画面を作る

## 状態

- 実装済み
- GitHub Issue: `#34`
- 最終更新: `2026-04-13`

## 何のための画面か

文字起こしのズレ、とくに固有名詞のズレを、人がすぐ確認して直せるようにする。

Pararia の品質の芯になる画面。

## この画面でやること

- 生の文字起こしを見る
- 推測された固有名詞候補を見る
- 正しいか間違いかを判断する
- 必要なら手で修正する
- 修正後の文字起こしを確認する
- そのままログ再生成に進む

## 画面に置くもの

### 上部

- 生徒名
- 面談 / 指導報告
- 実施日
- 現在の状態
  - 例: 要確認 / 確認済み / 再生成待ち
- `確認用文字起こしを更新` ボタン
- `この内容でログを作り直す` ボタン

### 左側

- 生の文字起こし
- 時系列に読める表示
- できれば発話ごと、区切りごとに見える表示

### 右側

- 確認が必要な候補一覧
- 各候補に出すもの
  - 元の文字
  - 推測した文字
  - なぜそう推測したか
  - 確信度
  - どこに出てきたか

### 各候補の操作

- 正しい
- 間違い
- 手入力で変更

### 下部

- 確認後の文字起こし
- 反映後の全文プレビュー
- 変更された箇所が分かる表示

## この画面で持つ状態

- 候補なし
- 候補あり
- 1件以上未確認
- 全件確認済み
- 再生成待ち
- エラー

## 大事なこと

- 全文を手で直させない
- 怪しいところだけ直せる
- 何が変わるかが分かる
- 直したらすぐ再生成に進める

## 完了条件

- 文字起こしの生データ、確認後データ、候補一覧を 1 画面で見比べられる
- 候補ごとの判断が迷わない
- 再生成へ進む導線が画面内で完結する
- 要確認が残っているかどうかを一目で判断できる
