# UI: 保護者レポート生成画面に、ログの信頼性が見える表示を足す

## 状態

- 実装済み
- GitHub Issue: `#38`
- 最終更新: `2026-04-13`

## 何のための画面か

どのログを親向けレポートに使ってよいか、安心して選べるようにする。

## この画面でやること

- ログを選ぶ
- まとまりを見る
- レポートを生成する
- 共有前に確認する

## 追加するもの

### 選択中ログごとに出すもの

- review 済み
- 要確認あり
- fallback 使用
- 固有名詞未確認あり
- 文字起こし未確認
- 信頼低め

### 追加候補に出すもの

- なぜ追加候補なのか
  - 情報が薄い
  - 次回確認が足りない
  - 保護者共有ポイントが弱い

### 生成前の注意

- このまま使ってよい
- 先に文字起こし確認をした方がよい
- 送付前にログを一度見た方がよい

## 今回入れた内容

- `ReportStudio` で選択中ログの件数と候補ログのまとまりを常時見えるようにした
- `buildBundleQualityEval()` による warning を selection / generated の両 view に表示するようにした
- 情報が薄いときに補うべき面談ログを `suggestedSessions` として追加候補に出すようにした
- preview 文言を `buildBundlePreview()` でまとめ、生成前に「この束で何が足りていて何が弱いか」を読めるようにした

## 大事なこと

- レポート生成が適当にログを混ぜる作業にならないこと
- 信頼できるログだけを使いやすくすること
- 画面の中で判断が完結すること

## 完了条件

- レポート生成前に、使うログの信頼状態が一目で分かる
- 確認が必要なログを無意識に選びにくい
- 先に見直すべきか、そのまま進めてよいかを画面が教えてくれる

## 確認

- `npm run typecheck`
- `npm run build`
