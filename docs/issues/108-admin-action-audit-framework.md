# admin write action の理由入力・確認・監査フレームワークを実装する

## 状態

- Done
- GitHub Issue: `#211`
- 作成日: `2026-04-26`

## 目的

ジョブ再実行、端末 revoke、ユーザー停止、PII 表示などの高権限操作を、理由入力・影響確認・監査つきで実行できる共通フレームワークにする。

## 背景

世界水準のSaaS運用では、運営者が何をしたかだけでなく、なぜしたか、何に影響したか、戻せるかを追跡できる必要がある。

## やること

- admin action request schema を定義する
- 理由入力を必須にする
- 操作対象と影響範囲を実行前に表示する
- admin write action を `PlatformAuditLog` に必ず保存する
- 成功 / 失敗 / キャンセルを監査できるようにする
- 危険操作用の確認 UI primitive を用意する

## 完了条件

- admin write 操作は監査なしで実行できない
- 理由なしで危険操作を実行できない
- 操作前に対象校舎と対象リソースを確認できる
- focused test で監査ログ保存を確認できる
