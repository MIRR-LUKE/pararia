# 会話ログを作る処理を、役割ごとに分けてわかりやすくする

## 状態

- 実装済み
- GitHub Issue: `#15`
- 最終更新: `2026-03-26`

## 何をするか

会話ログを作る処理を、1つの大きいファイルや大きい関数に寄せず、役割ごとに分ける。

特に、次の 2 つははっきり分ける。

- 面談ログを作る処理
- 指導報告ログを作る処理

## なぜやるか

今は処理が 1 か所に集まりやすく、少し直しただけでも別の出力が壊れやすい。

このままだと、

- prompt を直したら別モードが壊れる
- fallback を直したら整形結果も崩れる
- どこを直せばいいか毎回わかりにくい

という状態になりやすい。

## やること

- prompt 定義を分ける
- 生成を回す処理を分ける
- 整形や後処理を分ける
- fallback を別にする
- 面談モードと指導報告モードの違いをファイルの構造でも見えるようにする

## 終わったといえる状態

- [x] 会話ログを作る責務が小さく分かれている
- [x] 面談用と指導報告用の定義が別ファイルになっている
- [x] fallback の修正が他の処理に波及しにくい
- [x] 新しい生成要件を追加するときの修正場所がわかりやすい

## 今回入れたもの

- `lib/ai/conversation/` 配下へ `spec / generate / normalize / fallback / transport / shared / types` を分離した
- `lib/ai/conversationPipeline.ts` は互換用の薄い入口にした
- 面談と指導報告の spec を別定義にした
