# admin UX/UI の簡素化チェックと非エンジニア向けユーザビリティ確認を追加する

## 状態

- Done
- GitHub Issue: `#218`
- 作成日: `2026-04-26`

## 目的

platform admin を、エンジニア向けの複雑な監視画面ではなく、非エンジニア情シス・CSでも扱える業務画面として維持する。

## 背景

管理画面は放っておくと、カード、グラフ、タブ、内部コードが増えていく。仕様で決めた「超シンプル」を実装後も守るため、確認観点を issue として独立させる。

## やること

- admin 画面の UX/UI checklist を作る
- 初期表示 3 領域以内、主要カード 4 つ以内、表 6 列前後を確認する
- 内部コードや worker 名が初期表示に出ていないことを確認する
- キーボード操作、フォーカス、状態バッジ、空状態、権限不足状態を確認する
- 5 つのユーザビリティタスクを Playwright または手動手順として残す
- admin spec に checklist への導線を置く。README は今回の担当範囲外のため後続で更新する

## 完了条件

- 非エンジニアが `校舎検索 -> 詳細 -> 要対応確認` を迷わず実行できる
- 危険操作の誤クリックを防ぐ導線になっている
- UI が説明文だらけになっていない
- アクセシビリティの最低基準を満たす
- 実装レビュー時に使える checklist が残っている
- `npx tsx scripts/check-admin-console-simplification.ts` を package.json なしで単体実行できる
