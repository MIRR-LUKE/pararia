# 再生成の途中失敗で、既存の面談ログを消さない

## 何がつらいか

- いまは再生成ボタンを押した時点で、既存の `artifactJson` と `summaryMarkdown` を先に空にしている
- そのあとで review 再構築や job enqueue が失敗すると、元は読めていたログが空のままになる
- 生成まわりは不安定さが 1 回あるだけでデバッグ負担が大きく、ここは特に壊してはいけない

## 目標

- 再生成を始めても、最後に新しい成果物が完成するまでは「前回の正常なログ」を残す
- 再生成途中で失敗しても、講師は最低限いままでのログを読める
- 失敗時は「再生成は失敗したが、前回のログは残っている」と分かる状態にする

## やること

- 再生成開始時に `artifactJson` / `summaryMarkdown` / `formattedTranscript` を先に消さない
- 再生成ジョブは `status` と job キューだけを更新して始める
- 新しい成果物の保存が成功した時点でだけ、表示中のログを新しい内容へ差し替える
- 回帰テストを追加して、再生成開始直後に既存成果物が消えないことを確認する

## 完了条件

- 再生成開始後も既存ログが読める
- 再生成失敗時に既存ログが残る
- 専用の回帰テストが追加されている
