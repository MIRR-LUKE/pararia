# UI: 共有・送付まわりの運用画面を作る

## 状態

- 未着手
- GitHub Issue: `#40`
- 最終更新: `2026-03-27`

## 何のための画面か

保護者レポートを送ったあと、何が送れていて、何が失敗しているかを管理できるようにする。

## この画面でやること

- 送付済みを見る
- 手動共有を見る
- 失敗を見る
- 再送対象を見る

## 画面に置くもの

### 一覧

- 生徒名
- 作成日
- 送付状態
- 送付方法
- 最終更新
- 再送が必要か

### フィルター

- 送付済み
- 手動共有
- 送信失敗
- 宛先エラー
- 再送待ち

## 大事なこと

- レポート生成して終わりにならないこと
- 共有運用まで含めて SaaS にすること
- 送ったかどうかを曖昧にしないこと

## 完了条件

- 送付状態を一覧だけで把握できる
- 再送が必要なものをすぐ見つけられる
- 手動共有と自動送信が混ざっても追跡しやすい
