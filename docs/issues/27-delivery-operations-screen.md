# UI: 共有・送付まわりの運用画面を作る

## 状態

- 実装済み
- GitHub Issue: `#40`
- 最終更新: `2026-04-13`

## 何のための画面か

保護者レポートを送ったあと、何が送れていて、何が失敗しているかを管理できるようにする。

## この画面でやること

- 送付済みを見る
- 手動共有を見る
- 失敗を見る
- 再送対象を見る

## 画面に置くもの

### 一覧

- 生徒名
- 作成日
- 送付状態
- 送付方法
- 最終更新
- 再送が必要か

### フィルター

- 送付済み
- 手動共有
- 送信失敗
- 宛先エラー
- 再送待ち

## 今回入れた内容

- `/app/reports` で `送信済み / 手動共有 / 遅延` のカウントを上段に出した
- filter を `未作成 / review / 共有待ち / 送信済み / 手動共有 / 遅延` まで分けた
- 各生徒行で `source trace`, 最新の送付状態, 次のアクションをまとめて見えるようにした
- `遅延共有キュー` を別 block で出し、再送や手動確認が必要なものを拾いやすくした

## 大事なこと

- レポート生成して終わりにならないこと
- 共有運用まで含めて SaaS にすること
- 送ったかどうかを曖昧にしないこと

## 完了条件

- 送付状態を一覧だけで把握できる
- 再送が必要なものをすぐ見つけられる
- 手動共有と自動送信が混ざっても追跡しやすい
