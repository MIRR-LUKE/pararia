# 契約・CS・サポートメモの運営メタデータを実装する

## 状態

- Done
- GitHub Issue: `#215`
- 作成日: `2026-04-26`

## 目的

請求書払い前提の法人SaaSとして、校舎運用に必要な契約状態、担当者、サポートメモを platform admin で確認できるようにする。

## 背景

アプリ内課金やサブスク決済の実装は不要。一方で、契約状態や担当CSが分からないと、運営・CS・障害対応が属人化する。

## やること

- 校舎または顧客に契約メタデータを持たせる
- 契約状態、契約更新日、請求先、営業担当、CS担当、利用上限、特記事項を扱う
- サポートメモを校舎詳細に出す
- 初回は確認専用または最小編集に限定する
- 財務システムの代替にしない

## 2026-04-26 実装

- `Organization` に契約状態、契約更新日、請求先、営業担当、CS担当、利用上限メモ、サポートメモを追加した
- `/admin` の校舎一覧で契約状態、更新日、CS担当を確認できる
- `/admin/campuses/[organizationId]` の `契約・担当` と `メモ` で運営参照に必要な情報を表示する
- 初期版は確認専用に限定し、請求システムにはしない
- 更新操作を入れる場合は後続で `PlatformAuditLog` 監査を必須にする

## 完了条件

- 運営者が校舎の契約・担当・注意事項を確認できる
- 請求書払い前提の運用情報が admin に集約される
- 契約画面が請求システムになりすぎていない
- 更新操作を入れる場合は監査される
