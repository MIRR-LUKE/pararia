# `/admin` ルートを運営司令塔ダッシュボードに作り直す

## 状態

- Done
- GitHub Issue: `#207`
- 作成日: `2026-04-26`

## 目的

`/admin` の最初の画面を、何百校舎のうち「今日どこに対応すべきか」が一目で分かる司令塔にする。

## 背景

現 `/admin` は校舎内 `settings` 由来の保守情報を表示しており、全校舎横断の運営画面になっていない。初回実装では read-only でよいので、要対応と校舎検索を中心に作り直す。

## 2026-04-26 実装

- `/admin` を `PlatformOperator` 専用の全校舎 command center に置き換えた
- 主要指標を 4 つに制限し、初期表示は要対応一覧、処理状況、校舎検索に絞った
- 内部エラーコード、worker 名、面談本文、音声、保護者連絡先を初期表示に出さない構成にした

## やること

- `/admin` ルートを platform admin shell に置き換える
- 上部に `要対応`, `処理中`, `正常`, `停止中` の 4 状態だけを表示する
- メイン領域を `今日の要対応` 一覧にする
- 一覧列を `状態`, `校舎`, `何が起きているか`, `経過時間`, `担当`, `次にすること` に絞る
- SLO / p95 / 内部コードは初期表示に出さない
- 既存の運用保守部品は `/admin/operations` 相当に格下げする

## 完了条件

- `/admin` が校舎内設定の延長ではなく platform admin ホームになる
- 初期表示が 3 領域以内に収まっている
- 主要カードが 4 つ以内に収まっている
- 非エンジニアでも要対応校舎を特定できる
- 内部コードや worker 名が初期表示に出ない
