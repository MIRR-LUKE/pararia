# 横断ジョブヘルスと詰まり検出を実装する

## 状態

- Done
- GitHub Issue: `#210`
- 作成日: `2026-04-26`

## 目的

STT、面談ログ生成、配信、削除、Runpod 関連の詰まりを、校舎横断で早く見つけられるようにする。

## 背景

運営コンソールの価値は「どの処理が止まっていて、どの校舎に影響があるか」を素早く知れることにある。初回は read-only の要対応検出に限定する。

## やること

- 横断ジョブヘルス snapshot を追加する
- `要対応のみ` を初期表示にする
- 一覧列を `状態`, `校舎`, `処理`, `経過時間`, `原因`, `次にすること` に絞る
- STT / generation / delivery / storage deletion / Runpod の代表的な詰まりを分類する
- 内部 error code は詳細内に隠し、初期表示は日本語原因にする
- 校舎詳細から該当ジョブに drilldown できるようにする

## 完了条件

- 運営者が詰まった処理を校舎横断で把握できる
- 原因分類が日本語で表示される
- 正常終了済みジョブは初期表示に出ない
- 危険操作はまだ露出しない
