# 校舎ディレクトリ検索とフィルタを実装する

## 状態

- Done
- GitHub Issue: `#208`
- 作成日: `2026-04-26`

## 目的

運営者が何百校舎の中から、校舎名・顧客名・管理者メール・契約番号・校舎コードで目的の校舎をすぐ探せるようにする。

## 背景

全校舎管理では、最初に必要なのは分析グラフではなく検索である。初回実装では既存 `Organization` を `校舎` として扱い、`CustomerAccount` / `Campus` の本格移行は後続に分ける。

## 2026-04-26 実装

- `/api/admin/platform` に全校舎 read-only list API を追加した
- `/admin` に校舎名、契約名、プラン、ID の検索と状態フィルタを追加した
- 100 件単位の読み込みにし、何百校舎でも初期表示が重くなりすぎないようにした

## やること

- 全校舎横断の read-only list API を追加する
- 検索欄に初期フォーカスを置く
- 初期フィルタを `すべて`, `要対応`, `稼働中`, `導入準備中`, `停止中` に絞る
- 一覧列を `状態`, `校舎`, `顧客`, `生徒数`, `最終利用`, `担当`, `詳細` に絞る
- 詳細フィルタは `さらに絞り込む` に隠す
- ページングまたは cursor を入れ、何百校舎でも重くならないようにする

## 完了条件

- 運営者は `/admin` から全校舎を検索できる
- 初期表示で表の列が増えすぎない
- 校舎内 `organizationId` に暗黙依存しない
- 何も見つからない場合に、次に何を試すかが日本語で表示される
