# UI: ログ確認画面を、信頼できるか判断できる画面に作り直す

## 状態

- 実装済み
- GitHub Issue: `#35`
- 最終更新: `2026-04-13`

## 何のための画面か

生成されたログをただ読むだけでなく、このログをそのまま使ってよいか判断できるようにする。

## この画面でやること

- 生成されたログを見る
- 元の文字起こしを見る
- このログが確認済みかどうかを確かめる
- 信頼が低いときに気づける
- 必要なら文字起こし確認へ戻る

## 画面に置くもの

### 上部

- 生徒名
- 面談ログ / 指導報告ログ
- 状態
  - 例: 生成中 / 確認可能 / エラー
- review 状態
  - 例: 確認済み / 要確認あり
- fallback 使用の有無
- `文字起こし確認へ` ボタン

### タブ

- ログ本文
- 文字起こし
- 将来追加しやすい `根拠` タブの置き場

### ログ本文側

- セクションごとに整って見えるレイアウト
- 根拠あり / 推測あり / 要確認あり が分かる表示
- 次回確認事項
- 保護者共有向けの要点

### 文字起こし側

- 生文字起こし
- 確認後の文字起こし
- 必要なら切り替えて見られる表示

## この画面で持つ状態

- 生成中
- 生成完了
- エラー
- review 未完了
- fallback 使用
- 文字起こし未確認

## 大事なこと

- 綺麗なログではなく、使ってよいログか判断できること
- review の必要性にすぐ気づけること
- 文字起こし確認へ戻りやすいこと

## 完了条件

- ログ本文を見るだけで、信頼して使えるかの判断材料がそろっている
- 文字起こし確認に戻る導線が迷わない
- review 未完了や fallback 使用が埋もれない
