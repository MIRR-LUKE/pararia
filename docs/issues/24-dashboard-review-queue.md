# UI: ダッシュボードに「文字起こし確認待ち」を入れる

## 状態

- 未着手
- GitHub Issue: `#37`
- 最終更新: `2026-03-27`

## 何のための画面か

朝いちで、どの生徒の何を先に確認すべきかを分かるようにする。

## 追加するもの

### 指標

- 文字起こし確認待ち
- 固有名詞の未確認件数
- 低信頼ログ件数
- fallback 使用件数
- レポート作成前に見直すべき件数

### 優先キュー

今ある

- 面談を始める
- 授業を始める
- レポートを作る

に加えて

- 文字起こしを確認する
- ログを見直す

を入れる。

## 大事なこと

- 今日の作業が「録る」「作る」だけにならないこと
- 確認がちゃんと仕事として見えること

## 完了条件

- ダッシュボードを見た時点で、確認が必要な仕事量が分かる
- 確認待ちのタスクへ 1 クリックで移動できる
- 緊急度が低いものと高いものが見分けられる
