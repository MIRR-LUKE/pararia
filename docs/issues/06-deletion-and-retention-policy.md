# 削除ルールと保存期間のルールをはっきり決める

## 状態

- 実装済み
- GitHub Issue: `#19`
- 最終更新: `2026-03-26`

## 何をするか

録音、文字起こし、生成したログ、レポート、監査ログなどについて、

- 何を消せるのか
- 何を残すのか
- いつ消すのか

をルールとして決める。

## なぜやるか

教育系 SaaS では、データごとに扱いが違う。

たとえば、

- 録音は消したい
- 文字起こしは残すかもしれない
- 監査ログは簡単に消してはいけない

のように、同じ「削除」でも考え方が違う。

ここがあいまいだと、あとで運用事故になりやすい。

## やること

- データの種類ごとに削除ルールを決める
- soft delete か hard delete かを決める
- audit log をどう残すか決める
- 管理者がどこまで削除できるか決める
- 保存期間が必要なら期間も決める
- README か docs に明文化する

## 終わったといえる状態

- [x] データごとの削除方針が決まっている
- [x] API の挙動と運用ルールが一致している
- [x] 削除時の副作用が説明できる
- [x] 最低限必要な監査履歴が残る

## 今回入れたもの

- `docs/data-retention-policy.md` に削除 / 保存期間ルールを書いた
- cleanup API で期限切れの transcript / audio runtime file / report delivery event を整理するようにした
- 生徒削除時に runtime 上の録音ファイルや live manifest も削除するようにした
- audit log はアプリから削除しない方針を維持した
