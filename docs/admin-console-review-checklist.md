# Platform Admin レビュー checklist

最終更新: 2026-04-26

## 目的

この checklist は、`/admin` を PARARIA 運営側の platform admin として実装した後に、仕様から外れていないかをレビューするためのもの。

正本仕様は [admin-console-platform-spec.md](./admin-console-platform-spec.md)。

## レビュー対象

- `/admin` と `app/api/admin/**`
- admin サブドメイン guard
- PlatformOperator / PlatformRole / PlatformAuditLog
- admin から呼ばれる operations / support / audit API
- `/app/settings` から platform admin 相当の運営保守操作が露出していないこと

## 入口と権限

- [ ] 本番の正式入口は admin サブドメインにできる。
- [ ] 通常ホストの `/admin` は admin base URL に寄せられる。
- [ ] 校舎内 `ADMIN` だけでは `/admin` と `app/api/admin/**` に入れない。
- [ ] `PlatformOperator` が DB 上の正で、メール allowlist は移行・緊急用に限定されている。
- [ ] admin API は対象校舎または対象リソースを明示し、暗黙の `session.user.organizationId` に依存していない。

## 情報設計

- [ ] `/admin` 初期表示は 3 領域以内で、目的が `要対応の把握` に絞られている。
- [ ] ダッシュボードの主要カードは 4 つ以内。
- [ ] 初期表示のグラフは 1 つ以内。分析より要対応一覧が主役になっている。
- [ ] 表の初期列は 6 列前後で、詳細は行クリック後の詳細パネルに逃がしている。
- [ ] タブは必要最小限で、校舎詳細は `概要`, `ユーザー`, `ジョブ`, `端末`, `監査`, `メモ` を超えて増やさない。
- [ ] 初期表示に面談本文、音声、保護者連絡先、過剰な PII が出ていない。

## 文言と状態表示

- [ ] 画面名とラベルは業務の言葉になっている。`tenant` は `校舎`、`worker` は `処理基盤` として扱う。
- [ ] 初期表示に内部 error code、例外名、worker 名、DB カラム名が出ていない。
- [ ] エラー文言は `何が起きたか`, `影響`, `次にすること` の順に読める。
- [ ] 状態は色だけでなく `要対応`, `処理中`, `正常`, `保留`, `停止中` などの文字を持つ。
- [ ] disabled ボタンだけを置かず、押せない理由と解決方法が近くにある。

## 危険操作

- [ ] 初回 read-only フェーズでは、再実行、キャンセル、端末 revoke、ユーザー停止、代理閲覧、本文表示が常時露出していない。
- [ ] write 操作を入れる場合、理由入力が必須。
- [ ] write 操作前に対象校舎、対象リソース、影響範囲、戻せるかが確認できる。
- [ ] write 操作の成功、失敗、キャンセルが `PlatformAuditLog` に残る。
- [ ] 一括危険操作は初回では作らない。追加する場合は件数プレビュー、理由、承認、監査を必須にする。

## アクセシビリティ

- [ ] キーボードだけで検索、フィルタ、詳細確認、確認ダイアログまで進める。
- [ ] フォーカスリングが見える。
- [ ] テキストコントラストは WCAG 2.2 AA を満たす。
- [ ] 行アクションはスクリーンリーダー向けに対象名を含む。例: `田中校の詳細を見る`。
- [ ] 空状態、ローディング、エラー、権限不足の専用状態がある。

## 非エンジニア向け 5 タスク確認

レビュー時は、CS または非エンジニア情シス役の確認者が次の 5 タスクを実施する。

1. 校舎名で検索して詳細を開ける。
2. 要対応の校舎を見つけ、原因分類を日本語で説明できる。
3. 詰まったジョブを見つけ、再実行前に影響範囲を確認できる。
4. 端末を revoke する前に対象校舎と端末名を確認できる。
5. 監査ログから、誰がいつ何をしたか説明できる。

## 単体チェック補助

package.json は更新せず、必要なときだけ次を直接実行する。

```bash
npx tsx scripts/check-admin-console-simplification.ts
```

この script は静的な補助チェックであり、手動タスク確認と置き換えない。
