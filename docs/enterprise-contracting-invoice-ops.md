# エンタープライズ契約・請求書払い運用

最終更新: 2026-04-25

## 1. 目的

この文書は、PARARIA を日本国内の上場企業・学校法人・教育事業者へ直販する際の、契約、請求書払い、SLA、オンボード、オフボード、停止、更新の運用を定義する。

PARARIA は日本向け SaaS として、国内の稟議、学校法人の事務処理、上場企業の購買・法務・情報システム審査に耐える契約運用を行う。海外展開を前提にした決済最適化ではなく、**日本の法人契約に合わせた直販・請求書払いを基本としつつ、監査証跡、SLA、委託先説明は世界水準で整える**。

決済ゲートウェイ、アプリ内課金、クレジットカード自動課金、ストア課金は本運用の対象外とする。

## 2. 基本方針

- 契約は法人または学校法人単位で締結する。
- 利用開始前に、申込書、利用規約、個人情報取扱い、委託先説明、SLA、請求条件を合意する。
- 支払いは請求書払いを標準とする。
- 月額・年額・PoC・初期費用・サポート費用は見積書と注文書または申込書で明示する。
- 契約状態、請求状態、利用停止、更新期限は台帳で管理する。
- 未払い停止とセキュリティ停止は区別して扱う。

## 3. 契約形態

### 3.1 標準契約

| 項目 | 標準 |
| --- | --- |
| 契約主体 | 法人、学校法人、教育事業者 |
| 契約期間 | 12 か月 |
| 更新 | 自動更新または事前合意更新 |
| 支払い | 請求書払い |
| 請求単位 | 月額、年額、または契約で定める単位 |
| 支払期日 | 月末締め翌月末払い等、契約で定める |
| 利用範囲 | 組織、校舎、部署、利用者数、生徒数等で定義 |
| サポート | 平日営業時間内を標準 |

### 3.2 PoC / トライアル

- 期間、対象組織、対象データ、利用者、終了後削除を明記する。
- 本番個人データを使う場合は、有償契約と同等の安全管理措置を適用する。
- PoC 終了時は、正式契約へ移行するか、エクスポート・削除するかを決める。
- PoC 中の成果物、ログ、レポートの権利・削除条件を申込書に明記する。

### 3.3 個別契約・DPA

導入先から個別契約、覚書、DPA、情報セキュリティチェックシート、反社条項、秘密保持契約を求められた場合は、次を確認する。

- PARARIA が受託者として扱う個人データの範囲
- 再委託先の承諾方式
- 越境移転の説明方式
- 監査権、報告義務、インシデント通知期限
- SLA、損害賠償、責任制限
- 契約終了後のデータ返却・削除

## 4. 請求書払い

### 4.1 請求フロー

1. 見積書を発行する。
2. 導入先から注文書または申込書を受領する。
3. 契約開始日と請求開始日を確定する。
4. 請求書を発行する。
5. 入金予定日を台帳へ登録する。
6. 入金確認後、請求状態を更新する。
7. 未払いがある場合は督促、猶予、停止判断を記録する。

### 4.2 請求書記載項目

- 契約名
- 対象組織名
- 契約期間
- 請求対象期間
- プラン名または利用範囲
- 初期費用、月額費用、年額費用、オプション費用
- 消費税
- 支払期日
- 振込先
- 請求書番号
- 発行日

### 4.3 未払い対応

| 経過 | 対応 |
| --- | --- |
| 支払期日前 | 通常運用 |
| 支払期日超過 | 事務連絡、請求書再送 |
| 30 日超過 | 管理者へ督促、停止予告 |
| 60 日超過 | 新規作成停止またはログイン制限を検討 |
| 90 日超過 | 契約解除、データ保全、削除スケジュール協議 |

未払い対応では、ただちにデータを削除しない。停止範囲、解除条件、導入先への通知履歴を残す。

## 5. SLA

### 5.1 可用性

- 標準目標: 月間稼働率 99.5%
- 計画メンテナンスは可用性計算から除外できる。
- 外部サービス障害、導入先ネットワーク障害、不可抗力は個別契約に従う。

### 5.2 サポート時間

- 標準: 日本時間の平日営業時間内
- 重大障害は営業時間外でも検知時点から初動する運用を目指す。
- 個別契約により、優先サポート、専用窓口、月次レポートを追加できる。

### 5.3 重大度と応答目標

| 重大度 | 内容 | 初回応答目標 | 更新頻度 |
| --- | --- | --- | --- |
| Sev1 | 広範囲の利用不能、個人データ漏えい疑い | 4 営業時間以内 | 進展ごと、少なくとも日次 |
| Sev2 | 主要機能の限定的障害、特定組織の業務影響 | 1 営業日以内 | 必要に応じて |
| Sev3 | 軽微な不具合、問い合わせ | 3 営業日以内 | 通常サポート |

### 5.4 障害報告

障害報告には次を含める。

- 発生日時
- 検知日時
- 影響範囲
- 暫定原因
- 回避策
- 復旧日時
- 再発防止策
- データ影響の有無

## 6. オンボード

### 6.1 契約前

- 利用範囲、対象生徒数、対象校舎、対象部署を確認する。
- 個人情報、録音、保護者説明、学校法人規程、社内規程の確認状況を聞く。
- [privacy-compliance-pack.md](./privacy-compliance-pack.md) と [data-location-and-subprocessors.md](./data-location-and-subprocessors.md) を提供する。
- SLA、サポート、障害連絡、請求条件を合意する。

### 6.2 契約後

- tenant を作成する。
- 初期管理者を登録する。
- 初期データ投入の対象・方法・件数を決める。
- 研修日、初回利用日、検収日を決める。
- サポート窓口と障害連絡先を登録する。

### 6.3 初回検収

- ログインできること。
- 権限ごとの閲覧範囲が正しいこと。
- 録音、文字起こし、レビュー、レポート生成が動作すること。
- 誤送信防止、削除、アーカイブ、復旧手順を説明済みであること。
- 請求開始条件を満たしていること。

## 7. 更新

### 7.1 更新前確認

契約満了の 60 日前を目安に次を確認する。

- 契約更新意思
- 利用範囲の増減
- 生徒数、利用者数、校舎数、部署数の変更
- SLA、サポート、請求条件の変更
- 委託先・データ所在地説明の更新有無
- 未払い、障害、インシデント、例外運用の有無

### 7.2 更新証跡

- 更新見積書
- 注文書または更新申込書
- 契約期間
- 価格
- 請求開始日
- 更新説明を受けた担当者
- 内部承認者

## 8. 停止

### 8.1 停止種別

| 種別 | 例 | データ扱い |
| --- | --- | --- |
| 契約停止 | 期間満了、解約 | エクスポート・削除手順へ移行 |
| 未払い停止 | 長期未払い | データは保全し、利用を制限 |
| セキュリティ停止 | 漏えい疑い、不正利用 | 最小範囲で封じ込め、証跡保全 |
| 導入先依頼停止 | 校内判断、移管、休止 | 合意範囲で停止 |

### 8.2 停止時の通知

- 停止理由
- 停止予定日時
- 停止範囲
- 利用できなくなる機能
- データエクスポート可否
- 解除条件
- 問い合わせ先

## 9. オフボード

契約終了時の詳細なデータ手順は [data-lifecycle-operations.md](./data-lifecycle-operations.md) を参照する。

事務運用としては次を実施する。

1. 解約日、最終利用日、最終請求日を確定する。
2. 未払い、返金、日割り、違約金の有無を確認する。
3. エクスポート要否と受領者を確認する。
4. 管理者以外のログインを停止する。
5. エクスポート完了後、削除猶予期間を開始する。
6. 削除完了証跡を残す。
7. 請求台帳と契約台帳を終了状態に更新する。

## 10. 台帳

### 10.1 契約台帳

- 顧客名
- 契約 ID
- 契約開始日
- 契約終了日
- 更新条件
- 利用範囲
- SLA
- サポート条件
- データ削除条件
- 契約担当者

### 10.2 請求台帳

- 請求書番号
- 請求対象期間
- 金額
- 税額
- 支払期日
- 入金日
- 未払い日数
- 督促履歴
- 停止予告履歴

### 10.3 SLA 台帳

- 障害 ID
- 重大度
- 発生日時
- 初回応答日時
- 復旧日時
- 影響範囲
- SLA 対象有無
- 報告書提出日

## 11. 審査での回答例

### 11.1 決済ゲートウェイを使うか

使わない。PARARIA のエンタープライズ提供では、クレジットカード自動課金、アプリ内課金、決済ゲートウェイ連携を前提にしない。契約は直販で行い、請求書払い、注文書、申込書、個別契約により管理する。

### 11.2 契約終了後にデータはどうなるか

導入先と合意したエクスポート期間後、tenant データを削除する。バックアップ内のデータは復旧目的に限定して保持し、バックアップ保持期間の満了により削除される。削除証跡には対象、実施者、日時、例外を残す。

### 11.3 未払い時にデータは消えるか

消さない。未払い時は督促、停止予告、利用制限を段階的に行い、データは保全する。契約解除後は、契約終了時のオフボード手順に従う。
